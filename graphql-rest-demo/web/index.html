📡 REST API

Multiple Endpoints

REST returns complete objects. You get all fields even if you only need a few.

🔍 Key Differences

  • Over-fetching: REST returns all fields. GraphQL returns only what you request.
  • Multiple Requests: REST may need multiple endpoints. GraphQL needs one.
  • Type Safety: GraphQL has built-in schema validation. REST needs external docs.
  • Exploration: GraphQL supports introspection. REST relies on documentation.

📚 API Reference

REST Endpoints:
GET    /blogs?author=&tag=&q=     List all blogs (optional filters)
GET    /blogs/{id}               Get specific blog
POST   /blogs                    Create new blog
GET    /search?q={query}         Search blogs

GraphQL Endpoint:
POST   /graphql

Example GraphQL Queries:

# Get only titles
{
  blogs {
    title
  }
}

# Get filtered by author
{
  blogs(author: "Alice Johnson") {
    title
    content
    tags
  }
}

# Get specific blog
{
  blog(id: "abc123") {
    title
    author
    created_at
  }
}

# Create blog (mutation)
mutation {
  createBlog(
    title: "New Post"
    content: "Content here"
    author: "John Doe"
    tags: ["graphql", "demo"]
  ) {
    id
    title
    author
  }
}
